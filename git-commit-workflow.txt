Git workflow

Use this sequence when committing and pushing changes:

git status
git add -A
git commit -m "message"
git push

Commit messages

- Aim for roughly five to seven words.
- Describe what changed (not how you implemented it unless that is the change).
- Start with an imperative verb, for example: add, fix, update, remove, refactor, revert.

Assistant / prompt shorthand

When you want only the commands back (no commentary), ask for responses in this shape: return only the four Git commands above with an appropriate commit message filled in after -m.

There is room to interpret edge cases (scope of git add -A, branch flow, amended commits); this outline is the default habit.
